# ภาพรวมการเริ่มต้นใช้งาน

OpenClaw รองรับหลายเส้นทางการเริ่มต้นใช้งาน ขึ้นอยู่กับตำแหน่งที่ Gateway ทำงาน
และวิธีที่คุณต้องการกำหนดค่าผู้ให้บริการ

## เลือกเส้นทางการเริ่มต้นใช้งานของคุณ

- **CLI wizard** สำหรับ macOS, Linux และ Windows (ผ่าน WSL2)
- **macOS app** สำหรับการตั้งค่าแบบมีคำแนะนำในครั้งแรกบน Mac ที่ใช้ Apple silicon หรือ Intel

## ตัวช่วยเริ่มต้นใช้งาน CLI

รันตัวช่วยในเทอร์มินัล:

```bash
openclaw onboard
```

ใช้ CLI wizard เมื่อคุณต้องการควบคุม Gateway, เวิร์กสเปซ,
ช่องทาง (channels) และ skills ได้อย่างเต็มที่ เอกสาร:

- [Onboarding Wizard (CLI)](/start/wizard)
- [`openclaw onboard` command](/cli/onboard)

## การเริ่มต้นใช้งานผ่านแอป macOS

ใช้แอป OpenClaw เมื่อคุณต้องการการตั้งค่าแบบมีคำแนะนำทั้งหมดบน macOS เอกสาร:

- [Onboarding (macOS App)](/start/onboarding)

## ผู้ให้บริการแบบกำหนดเอง

หากคุณต้องการ endpoint ที่ไม่ได้อยู่ในรายการ รวมถึงผู้ให้บริการแบบโฮสต์ที่
รองรับ API มาตรฐานของ OpenAI หรือ Anthropic ให้เลือก **Custom Provider** ใน
CLI wizard คุณจะถูกขอให้:

- เลือกแบบที่เข้ากันได้กับ OpenAI, เข้ากันได้กับ Anthropic หรือ **Unknown** (ตรวจจับอัตโนมัติ)
- กรอก base URL และ API key (หากผู้ให้บริการกำหนดให้ต้องใช้)
- ระบุ model ID และ alias (ถ้ามี)
- เลือก Endpoint ID เพื่อให้สามารถมี custom endpoint หลายรายการพร้อมกันได้

สำหรับขั้นตอนโดยละเอียด ให้ทำตามเอกสารการเริ่มต้นใช้งาน CLI ด้านบน
