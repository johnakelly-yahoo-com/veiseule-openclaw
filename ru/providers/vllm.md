# vLLM

vLLM может обслуживать open-source (и некоторые пользовательские) модели через **OpenAI-совместимый** HTTP API. OpenClaw может подключаться к vLLM, используя API `openai-completions`.

OpenClaw также может **автоматически обнаруживать** доступные модели из vLLM, если вы включите эту возможность с помощью `VLLM_API_KEY` (подойдёт любое значение, если на вашем сервере не настроена аутентификация) и не укажете явную запись `models.providers.vllm`.

## Быстрый старт

1. Запустите vLLM с OpenAI-совместимым сервером.

Ваш базовый URL должен предоставлять эндпоинты `/v1` (например, `/v1/models`, `/v1/chat/completions`). Обычно vLLM запускается по адресу:

- `http://127.0.0.1:8000/v1`

2. Включите (подойдёт любое значение, если аутентификация не настроена):

```bash
export VLLM_API_KEY="vllm-local"
```

3. Выберите модель (замените на один из ваших идентификаторов модели vLLM):

```json5
{
  agents: {
    defaults: {
      model: { primary: "vllm/your-model-id" },
    },
  },
}
```

## Обнаружение моделей (неявный провайдер)

Когда задан `VLLM_API_KEY` (или существует профиль аутентификации) и вы **не** определяете `models.providers.vllm`, OpenClaw выполнит запрос:

- `GET http://127.0.0.1:8000/v1/models`

…и преобразует возвращённые идентификаторы в записи моделей.

Если вы явно задаёте `models.providers.vllm`, автообнаружение отключается, и модели необходимо определить вручную.

## Явная конфигурация (модели вручную)

Используйте явную конфигурацию, если:

- vLLM запущен на другом хосте/порту.
- Вы хотите зафиксировать значения `contextWindow`/`maxTokens`.
- Ваш сервер требует реальный API-ключ (или вы хотите управлять заголовками).

```json5
{
  models: {
    providers: {
      vllm: {
        baseUrl: "http://127.0.0.1:8000/v1",
        apiKey: "${VLLM_API_KEY}",
        api: "openai-completions",
        models: [
          {
            id: "your-model-id",
            name: "Local vLLM Model",
            reasoning: false,
            input: ["text"],
            cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 },
            contextWindow: 128000,
            maxTokens: 8192,
          },
        ],
      },
    },
  },
}
```

## Устранение неполадок

- Проверьте, доступен ли сервер:

```bash
curl http://127.0.0.1:8000/v1/models
```

- Если запросы завершаются ошибками аутентификации, задайте реальный `VLLM_API_KEY`, соответствующий конфигурации вашего сервера, или явно настройте провайдера в `models.providers.vllm`.
