# Мастер онбординга (CLI)

CLI‑мастер онбординга — рекомендуемый способ настройки OpenClaw на macOS, Linux и Windows (через WSL2). Он настраивает локальный Gateway или подключение к удалённому Gateway, а также конфигурирует параметры рабочего пространства по умолчанию, каналы и Skills.

```bash
openclaw onboard
```

<Info>
Самый быстрый способ начать первый чат: откройте Control UI (настройка каналов не требуется). Выполните `openclaw dashboard` и общайтесь в браузере. Документация: [Dashboard](/web/dashboard).
</Info>

## Быстрый старт vs детальная настройка

Мастер начинает с выбора **Быстрого старта** (настройки по умолчанию) или **Детальной настройки** (полный контроль).

<Tabs>
  <Tab title="クイックスタート（デフォルト設定）">
    - Локальный Gateway на loopback
    - Существующее рабочее пространство или рабочее пространство по умолчанию
    - Порт Gateway `18789`
    - Токен аутентификации Gateway генерируется автоматически (создаётся даже на loopback)
    - Публичный доступ через Tailscale отключён
    - Личные сообщения Telegram и WhatsApp по умолчанию работают по списку разрешённых (может потребоваться ввод номера телефона)
  
</Tab>
  <Tab title="詳細設定（完全な制御）">
    - Отображает полный поток промптов для режима, рабочего пространства, Gateway, каналов, демонов и Skills
  
</Tab>
</Tabs>

## Часто используемые последующие команды

```bash
openclaw configure
openclaw agents add <name>
```

<Note>
`--json` не означает неинтерактивный режим. В скриптах используйте `--non-interactive`.
</Note>

<Tip>
Рекомендуется: настройте API-ключ Brave Search, чтобы агент мог использовать `web_search` (`web_fetch` работает без ключа). Самый простой способ: выполните `openclaw configure --section web`, чтобы сохранить `tools.web.search.apiKey`. Документация: [Webツール](/tools/web).
</Tip>
