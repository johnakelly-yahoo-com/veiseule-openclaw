# Обзор онбординга

OpenClaw поддерживает несколько сценариев онбординга в зависимости от того, где запущен Gateway
и как вы предпочитаете настраивать провайдеров.

## Выберите сценарий онбординга

- **CLI-мастер** для macOS, Linux и Windows (через WSL2).
- **Приложение macOS** для пошагового первого запуска на Apple silicon или Intel Mac.

## CLI-мастер онбординга

Запустите мастер в терминале:

```bash
openclaw onboard
```

Используйте CLI-мастер, если вам нужен полный контроль над Gateway, рабочим пространством,
каналами и навыками. Документация:

- [Onboarding Wizard (CLI)](/start/wizard)
- [`openclaw onboard` command](/cli/onboard)

## Онбординг через приложение macOS

Используйте приложение OpenClaw, если вам нужна полностью пошаговая настройка на macOS. Документация:

- [Onboarding (macOS App)](/start/onboarding)

## Пользовательский провайдер

Если вам нужен endpoint, которого нет в списке, включая размещённых провайдеров,
предоставляющих стандартные API OpenAI или Anthropic, выберите **Custom Provider** в
CLI-мастере. Вам будет предложено:

- Выбрать совместимость с OpenAI, совместимость с Anthropic или **Unknown** (автоопределение).
- Ввести базовый URL и API-ключ (если требуется провайдером).
- Указать ID модели и необязательный псевдоним.
- Выбрать Endpoint ID, чтобы несколько пользовательских endpoint могли сосуществовать.

Для подробных шагов следуйте документации по CLI-онбордингу выше.
