# آن بورڈنگ کا جائزہ

OpenClaw متعدد آن بورڈنگ راستوں کی سہولت دیتا ہے، اس بات پر منحصر ہے کہ Gateway کہاں چل رہا ہے
اور آپ پرووائیڈرز کو کس طرح کنفیگر کرنا پسند کرتے ہیں۔

## اپنا آن بورڈنگ راستہ منتخب کریں

- **CLI وزرڈ** macOS، Linux، اور Windows (بذریعہ WSL2) کے لیے۔
- **macOS ایپ** Apple silicon یا Intel Macs پر رہنمائی کے ساتھ پہلی بار چلانے کے لیے۔

## CLI آن بورڈنگ وزرڈ

ٹرمنل میں وزرڈ چلائیں:

```bash
openclaw onboard
```

جب آپ Gateway، ورک اسپیس، چینلز اور skills پر مکمل کنٹرول چاہتے ہوں تو CLI وزرڈ استعمال کریں۔ دستاویزات:

- [Onboarding Wizard (CLI)](/start/wizard)
- [`openclaw onboard` کمانڈ](/cli/onboard)

## macOS ایپ آن بورڈنگ

جب آپ macOS پر مکمل رہنمائی کے ساتھ سیٹ اپ چاہتے ہوں تو OpenClaw ایپ استعمال کریں۔ دستاویزات:

- [Onboarding (macOS App)](/start/onboarding)

## کسٹم پرووائیڈر

اگر آپ کو ایسا اینڈ پوائنٹ درکار ہے جو فہرست میں موجود نہیں، بشمول وہ ہوسٹڈ پرووائیڈرز جو معیاری OpenAI یا Anthropic APIs فراہم کرتے ہیں، تو CLI وزارڈ میں **Custom Provider** منتخب کریں۔ آپ سے درج ذیل کرنے کو کہا جائے گا:

- OpenAI-compatible، Anthropic-compatible، یا **Unknown** (auto-detect) منتخب کریں۔
- ایک بیس URL اور API key درج کریں (اگر پرووائیڈر کو درکار ہو)۔
- ایک ماڈل ID اور اختیاری عرف (alias) فراہم کریں۔
- ایک Endpoint ID منتخب کریں تاکہ متعدد کسٹم اینڈ پوائنٹس بیک وقت موجود رہ سکیں۔

تفصیلی مراحل کے لیے اوپر دی گئی CLI onboarding دستاویزات دیکھیں۔
