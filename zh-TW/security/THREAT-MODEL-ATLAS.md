## 1. 簡介

### 1.1 目的

本威脅模型使用專為 AI/ML 系統設計的 MITRE ATLAS 框架，記錄 OpenClaw AI 代理平台與 ClawHub 技能市集所面臨的對抗性威脅。

### 1.2 範圍

| 元件                   | 是否納入 | 備註                                              |
| ---------------------- | -------- | ------------------------------------------------- |
| OpenClaw Agent Runtime | 是       | 核心代理執行、工具呼叫、會話管理                  |
| Gateway                | 是       | 驗證、路由、通道整合                              |
| Channel Integrations   | 是       | WhatsApp、Telegram、Discord、Signal、Slack 等     |
| ClawHub Marketplace    | 是       | 技能發佈、審核、分發                              |
| MCP Servers            | 是       | 外部工具提供者                                    |
| User Devices           | 部分     | 行動應用程式、桌面客戶端                          |

### 1.3 不在範圍內

本威脅模型未明確排除任何範圍。

### 2.2 資料流程

| 流程 | 來源    | 目的地      | 資料               | 保護機制             |
| ---- | ------- | ----------- | ------------------ | -------------------- |
| F1   | Channel | Gateway     | 使用者訊息         | TLS、AllowFrom       |
| F2   | Gateway | Agent       | 路由後訊息         | 會話隔離             |
| F3   | Agent   | Tools       | 工具呼叫           | 政策強制執行         |
| F4   | Agent   | External    | web_fetch 請求     | SSRF 阻擋            |
| F5   | ClawHub | Agent       | 技能程式碼         | 審核、掃描           |
| F6   | Agent   | Channel     | 回應               | 輸出過濾             |

## 3. 依 ATLAS 戰術分類的威脅分析

### 3.1 偵察（AML.TA0002）

#### T-RECON-001：代理端點發現

| 屬性                    | 值                                                                   |
| ----------------------- | -------------------------------------------------------------------- |
| **ATLAS ID**            | AML.T0006 - 主動掃描                                                 |
| **說明**                | 攻擊者掃描暴露的 OpenClaw gateway 端點                               |
| **攻擊向量**            | 網路掃描、shodan 查詢、DNS 列舉                                      |
| **影響元件**            | Gateway、對外 API 端點                                               |
| **現有緩解措施**        | 可選 Tailscale 驗證、預設綁定 loopback                               |
| **殘餘風險**            | 中 - 公開 gateway 可被發現                                           |
| **建議**                | 文件化安全部署方式，對探索端點加入速率限制                           |

#### T-RECON-002：通道整合探測

| 屬性                    | 值                                                                 |
| ----------------------- | ------------------------------------------------------------------ |
| **ATLAS ID**            | AML.T0006 - 主動掃描                                               |
| **說明**                | 攻擊者探測訊息通道以識別由 AI 管理的帳號                           |
| **攻擊向量**            | 發送測試訊息、觀察回應模式                                         |
| **影響元件**            | 所有通道整合                                                        |
| **現有緩解措施**        | 無特定措施                                                          |
| **殘餘風險**            | 低 - 單純發現價值有限                                               |
| **建議**                | 考慮回應時間隨機化                                                 |
