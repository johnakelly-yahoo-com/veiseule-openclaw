### 29. 新增威脅

發現我們尚未涵蓋的攻擊向量或風險？ 31. 在 [openclaw/trust](https://github.com/openclaw/trust/issues) 開啟一個 issue，並用你自己的話描述。 32. 你不需要了解任何框架或填寫每個欄位——只要描述情境即可。

- 36. 你認為嚴重程度為何（低 / 中 / 高 / 致命）

### 建議緩解措施

有想法可以解決既有威脅嗎？ 43. 開啟一個 issue 或 PR 並引用該威脅。 44. 有效的緩解措施應具體且可執行——例如，「在 gateway 進行每個傳送者每分鐘 10 則訊息的速率限制」比「實作速率限制」更好。

### 提出攻擊鏈

攻擊鏈展示多個威脅如何結合成一個真實的攻擊情境。 47. 若你看到危險的組合，請描述步驟以及攻擊者會如何將它們串連。 48. 描述攻擊在實務中如何展開的簡短敘事，比正式範本更有價值。

### 修正或改進既有內容

錯字、釐清說明、過時資訊、更好的範例——歡迎 PR，無需先開 issue。

## 我們使用的內容

### MITRE ATLAS

此威脅模型建立於 [MITRE ATLAS](https://atlas.mitre.org/)（AI 系統的對抗性威脅版圖），這是一個專為 AI/ML 威脅（如提示注入、工具濫用與代理人利用）所設計的框架。 4. 你不需要了解 ATLAS 也能貢獻——我們會在審查期間將提交內容對應到該框架。

### 威脅 ID

每個威脅都會取得一個像是 `T-EXEC-003` 的 ID。 7. 類別如下：

| 程式碼     | 40. 類別 |
| ------- | ----------------------------- |
| RECON   | 偵察（Reconnaissance）— 資訊蒐集      |
| ACCESS  | 初始存取 — 取得進入權限                 |
| EXEC    | 執行 — 執行惡意行為                   |
| PERSIST | 持久化 — 維持存取權限                  |
| EVADE   | 防禦規避 — 避免被偵測                  |
| DISC    | 探索 — 了解環境                     |
| EXFIL   | 資料外洩 — 竊取資料                   |
| IMPACT  | 影響 — 造成損害或中斷                  |

ID 由維護者在審查期間指派。 27. 你不需要自行選擇一個。

### 風險等級

| 等級               | 30. 含義 |
| ---------------- | ----------------------------- |
| **嚴重（Critical）** | 完整系統遭到入侵，或高發生機率 + 嚴重影響        |
| **高（High）**      | 可能造成重大損害，或中等發生機率 + 嚴重影響       |
| **中（Medium）**    | 中度風險，或低發生機率 + 高影響             |
| **低（Low）**       | 發生機率低且影響有限                    |

若你不確定風險等級，只需描述影響，我們會進行評估。

## 審查流程

1. **分流（Triage）** — 我們會在 48 小時內審查新的提交內容
2. **評估（Assessment）** — 驗證可行性、指派 ATLAS 對應與威脅 ID、驗證風險等級
3. **文件化（Documentation）** — 確保所有內容格式正確且完整
4. **合併（Merge）** — 新增至威脅模型與視覺化
