# نظرة عامة على الإعداد الأولي

يدعم OpenClaw عدة مسارات للإعداد الأولي حسب مكان تشغيل Gateway وكيف تفضّل تهيئة المزودين.

## اختر مسار الإعداد الأولي الخاص بك

- **معالج CLI** لأنظمة macOS وLinux وWindows (عبر WSL2).
- **تطبيق macOS** لتجربة تشغيل أولى موجهة على أجهزة Apple silicon أو Intel Macs.

## معالج الإعداد الأولي عبر CLI

شغّل المعالج في الطرفية:

```bash
openclaw onboard
```

استخدم معالج CLI عندما تريد تحكمًا كاملاً في Gateway ومساحة العمل والقنوات والمهارات. المستندات:

- [Onboarding Wizard (CLI)](/start/wizard)
- [`openclaw onboard` command](/cli/onboard)

## الإعداد الأولي عبر تطبيق macOS

استخدم تطبيق OpenClaw عندما تريد إعدادًا موجهًا بالكامل على macOS. المستندات:

- [Onboarding (macOS App)](/start/onboarding)

## مزود مخصص

إذا كنت بحاجة إلى نقطة نهاية غير مدرجة، بما في ذلك المزودين المستضافين الذين يوفّرون واجهات OpenAI أو Anthropic القياسية، فاختر **Custom Provider** في معالج CLI. سيُطلب منك:

- اختيار متوافق مع OpenAI أو متوافق مع Anthropic أو **Unknown** (اكتشاف تلقائي).
- إدخال عنوان URL الأساسي ومفتاح API (إذا كان المزود يتطلب ذلك).
- توفير معرف نموذج واسم مستعار اختياري.
- اختيار معرف نقطة نهاية حتى يمكن لعدة نقاط نهاية مخصصة التعايش معًا.

للحصول على خطوات مفصلة، اتبع مستندات الإعداد الأولي عبر CLI أعلاه.
