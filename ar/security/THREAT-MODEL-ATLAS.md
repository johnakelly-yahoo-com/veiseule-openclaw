يعتمد نموذج التهديد هذا على [MITRE ATLAS](https://atlas.mitre.org/)، وهو الإطار القياسي في الصناعة لتوثيق التهديدات العدائية لأنظمة الذكاء الاصطناعي/تعلّم الآلة. تتم صيانة ATLAS بواسطة [MITRE](https://www.mitre.org/) بالتعاون مع مجتمع أمن الذكاء الاصطناعي.

**موارد ATLAS الرئيسية:**

- [ATLAS على GitHub](https://github.com/mitre-atlas/atlas-data)
- [المساهمة في ATLAS](https://atlas.mitre.org/resources/contribute)

هذه وثيقة حيّة يديرها مجتمع OpenClaw. راجع [CONTRIBUTING-THREAT-MODEL.md](./CONTRIBUTING-THREAT-MODEL.md) للاطلاع على إرشادات المساهمة:

## 1. المقدمة

### 1.1 الغرض

يوثق نموذج التهديد هذا التهديدات العدائية التي تستهدف منصة وكلاء الذكاء الاصطناعي OpenClaw وسوق المهارات ClawHub، باستخدام إطار MITRE ATLAS المصمم خصيصًا لأنظمة الذكاء الاصطناعي/تعلّم الآلة.

### 1.2 النطاق

| المكوّن               | مشمول | ملاحظات                                              |
| --------------------- | ------ | ---------------------------------------------------- |
| بيئة تشغيل وكيل OpenClaw | نعم    | تنفيذ الوكيل الأساسي، استدعاء الأدوات، الجلسات       |
| Gateway               | نعم    | المصادقة، التوجيه، تكامل القنوات                     |
| تكامل القنوات         | نعم    | WhatsApp، Telegram، Discord، Signal، Slack، وغيرها   |
| سوق ClawHub           | نعم    | نشر المهارات، الإشراف، التوزيع                       |
| خوادم MCP             | نعم    | مزوّدو الأدوات الخارجيون                             |
| أجهزة المستخدم        | جزئيًا | تطبيقات الجوال، عملاء سطح المكتب                     |

### 1.3 خارج النطاق

لا يوجد ما هو مستثنى صراحةً من نطاق نموذج التهديد هذا.

## 3. تحليل التهديدات حسب تكتيك ATLAS

### 3.1 الاستطلاع (AML.TA0002)

#### T-RECON-001: اكتشاف نقاط نهاية الوكيل

| الخاصية                | القيمة                                                               |
| ---------------------- | -------------------------------------------------------------------- |
| **معرّف ATLAS**        | AML.T0006 - الفحص النشط                                              |
| **الوصف**              | يقوم المهاجم بفحص نقاط نهاية Gateway المكشوفة لـ OpenClaw          |
| **متجه الهجوم**        | فحص الشبكة، استعلامات shodan، تعداد DNS                            |
| **المكوّنات المتأثرة** | Gateway، نقاط نهاية API المكشوفة                                    |
| **إجراءات التخفيف الحالية** | خيار مصادقة Tailscale، الربط بالحلقة المحلية افتراضيًا          |
| **المخاطر المتبقية**   | متوسطة - يمكن اكتشاف البوابات العامة                               |
| **التوصيات**           | توثيق النشر الآمن، إضافة تحديد معدل على نقاط الاكتشاف              |

#### T-RECON-002: استكشاف تكامل القنوات

| الخاصية                | القيمة                                                             |
| ---------------------- | ------------------------------------------------------------------ |
| **معرّف ATLAS**        | AML.T0006 - الفحص النشط                                            |
| **الوصف**              | يقوم المهاجم باختبار قنوات المراسلة لتحديد الحسابات المدارة بالذكاء الاصطناعي |
| **متجه الهجوم**        | إرسال رسائل اختبار، مراقبة أنماط الاستجابة                      |
| **المكوّنات المتأثرة** | جميع تكاملات القنوات                                              |
| **إجراءات التخفيف الحالية** | لا يوجد بشكل محدد                                              |
| **المخاطر المتبقية**   | منخفضة - قيمة محدودة من الاكتشاف وحده                            |
| **التوصيات**           | النظر في عشوائية توقيت الاستجابات                                 |
